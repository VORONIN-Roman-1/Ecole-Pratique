Contexte : Dans le cadre du stage de fin de cours, notre équipe de 3 personnes a décidé de faire le guide de l'univers Marvel. C’est un site web utilisant le pattern MVC regroupant des parties frontend, backend et une base de données. La fonctionnalité inclut la base des films et des personnages, la recherche adaptative, les liaisons many-to-many entre les films et les personnages. Nous sommes en train d’ajouter l’authentification des utilisateurs et le système des rôles pour que les visiteurs du site puissent ajouter des nouveaux éléments et/ou modifier des éléments existants.

Description du projet : Le projet utilise Java 8 et le framework Spring Boot pour la partie backend. Le backend fournit un service web avec le REST API , utilisant JSON comme format des données. La partie backend assure la recherche dans la base de données avec MySQL comme SGBDR et la création des objets JSON complexes qui regroupent l’information de plusieurs tables (relation many-to-many). Le ORM utilisé est Hibernate. Le build se fait avec Maven, et on a l’intention d’ajouter les tests unitaires Junit qui seront lancés automatiquement après le build.
Nous sommes en train d’ajouter l’authentification des utilisateurs OAuth avec Spring Security.
La partie frontend utilise Angular 16 qui prend le contenu depuis le service RESTful. Nous avons implémenté une application page unique qui utilise plusieurs modules et composants, directives, services, routing. Afin de se connecter au backend, on utilise le class HttpClient.
Le modèle des données est initialisé avec Hibernate et ensuite est rempli avec des requêtes SQL lancés à la main. La tâche d’automatisation est planifiée pour le sprint suivant.

Résultats : Sortie de la première version fonctionnelle du site, 2 sprints terminés, expérience de travail d’équipe agile obtenu.
